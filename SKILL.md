# SMART原则目标设定技能

## 定位与边界

本技能是**通用 SMART 原则工具**，服务于一切需要"把模糊意图变成可执行目标"的场景：
- 个人发展、团队管理、项目目标、质量改进、学习规划、业务增长……均为其**应用领域**之一。
- **不强制锚定任何行业标准（如 ISO 9001）**。质量管理只是本技能的一个应用方向，不是前提。
- 与 `qms-objective-decomposition`（质量目标分解）是**交集关系**：本技能是通用的 SMART 方法论超集；质量目标分解技能在质量场景下调用本技能的 SMART 能力。通用 SMART 优化详见 `smart-goal-setter`。

### 我能做什么
- 将模糊/愿景式目标改写为符合 SMART 五维度的清晰表述
- 生成关键结果（KRs）与分阶段行动计划
- 识别目标中的常见陷阱（模糊、不可量化、不切实际、无战略关联、无时限等）并给出修正建议
- 对已有目标做 SMART 五维机检（调用 `scripts/smart_validator.py`）

### 我不能做什么
- 不替代真实的目标管理工作（目标取舍、资源调配由用户决策）
- 不保证目标一定能达成（仅基于给定信息做可行性提示）
- 不预设行业标准对口（如非用户主动提供质量方针/战略，不做 ISO 等条款校验）
- 不替用户做业务判断（目标优先级、战略选择由用户定）

## 调用前置说明（规则 1）

使用前请明确以下输入：
- **goal_raw（必需）**：初始目标描述，可模糊
- **goal_type（建议）**：个人发展 / 团队管理 / 项目 / 质量改进 / 学习 / 业务 等（用于调整分析重点）
- **context（可选）**：背景、资源、限制
- **strategy（可选）**：上层战略/目标/其他关联目标——提供后校验 R 维相关性；**不提供则 R 维标注「待提供战略后复核」，不卡流程**

> 若上下文已含上述信息（如对话历史中用户已说过），可直接复用，无需重复询问；若缺失且影响结果质量，会主动提示并标注「供参考 / 待确认」。

## 输入载体判断（规则 2）

- 用户直接文本描述 → 直接解析
- 上传 `.md/.txt` → 读取
- 上传 `.docx` → 调用 docx 技能解析
- 上传 `.xlsx/.csv` → 调用 markitdown 技能解析为表格后抽取目标行
解析后统一抽取为「目标清单」再进入五步流程。

## 使用流程（五步）

### 步骤 1：收集目标信息
按"调用前置说明"引导或抽取输入。缺失非必需项时标注「供参考/待确认」。

### 步骤 2：SMART 五维分析
逐维分析（详见 `references/smart_principles.md`）：
- **S 具体**：5W1H 是否齐备、是否用具体动词、是否避开口号式模糊词
- **M 可衡量**：是否有量化指标 + 基准（建议"从 X 到 Y"）
- **A 可达成**：资源/能力/条件是否匹配（既有挑战又不脱离实际）
- **R 相关**：是否与用户给定的战略/上下文/其他目标对齐（无战略则标待复核）
- **T 时限**：是否有明确期限与里程碑

### 步骤 3：生成优化后的 SMART 目标
一句话概括，含全部 SMART 要素。

### 步骤 4：制定关键结果与行动计划
- KRs：3-5 个可量化结果，各含衡量标准
- 行动计划：阶段 / 时间 / 任务 / 交付物
- 自检清单：五维勾验

### 步骤 5：机检与陷阱识别
调用 `scripts/smart_validator.py` 对目标做五维机检；对照 `references/checklist.md`（五维自检 + 7 大陷阱）与 `references/traps.md`（速查表）识别风险。

## 缺失信息标注规范（规则 3）

凡输入缺失且影响结果，统一标注：
- 影响不大的 → 行内注「供参考」
- 关键缺失 → 行内注「待确认：需补充 XX 后方可定稿」

## 输出形式与大纲确认（规则 4）

- 默认输出：**文字版**结构化结果（五维表 + KRs + 行动计划 + 陷阱提示），直接在对话内给出。
- 若用户要求生成文档（.md/.docx），**必须先给出文档大纲**（见 `assets/SMART目标_输出模板.md`），用户确认结构后再生成，不先写全文。

## 资源索引

## 使用示例

### 示例：质量改进目标（本技能应用之一）
用户输入：
- goal_raw：我们要提高产品质量
- goal_type：质量改进
- context：缺陷率高、客户投诉多，期望半年改善

执行：识别"提高"为模糊词 → 改写为"将缺陷率从 2.5% 降至 1.5% 以内、投诉降 50%" → 生成 KRs 与行动计划 → 机检五维。

### 示例：个人发展目标（同为应用之一，与质量无关）
用户输入 goal_raw：我想提升职业竞争力 → 按个人发展类型生成晋升路径目标，无需任何质量/ISO 前提。

## 相关技能联动（战略分析工具链）

SMART 是工具链的**目标落成环节**：把战略意图转化为可衡量目标。完整链路：

`pestel-analysis`（宏观外部）→ `porters-five-forces`（行业外部）→ `vrio-analysis`（内部）→ `swot-analysis`（收敛）→ **`smart-goal-setter`（落成目标）** → `qms-objective-decomposition`（质量分解落地）

- **上游来源**：
  - `swot-analysis` 的 SO/WO/ST/WT 战略建议 → 用本技能落成 SMART 目标
  - 任意模糊意图（个人/团队/项目/质量/业务）——本技能通用，不限来源
- **下游/交集**：
  - `qms-objective-decomposition`（质量目标分解）：本技能是其内置 SMART 校验的通用超集；质量场景下二者交集，通用 SMART 优化详见本技能（交集关系已在 qms 技能中声明）
- **关系说明**：SMART 不绑定任何前置分析工具，但作为"战略 → 目标"的标准转化器，最常接在 SWOT 之后使用。
